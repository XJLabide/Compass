## Description
First-run seed: when a signed-in user has no `profile` doc, create `profile` (defaults), `program/active` (Upper/Lower 4-day template), and the master `exercises` list (~25 common lifts). Idempotent.

**Size:** M
**Files:** `src/lib/db/seed.ts`, `src/lib/data/upperLowerProgram.ts`, `src/lib/data/exerciseMaster.ts`

## Approach
- Trigger: `AuthProvider` calls `ensureSeeded(user)` after auth resolves and user is allowlisted.
- Idempotent: first reads `profile` — if exists, exit.
- Default `profile`: `unitSystem: 'imperial'`, `proteinTargetG: 180`, `weeklyGainKg: 0.2`, `timezone: Intl.DateTimeFormat().resolvedOptions().timeZone`.
- Template (Upper/Lower 4-day): Upper A, Lower A, Upper B, Lower B with 5–6 planned exercises each, target sets + rep ranges (e.g. 4x6–10).
- Exercise master: bench, OHP, row, pull-up, lat pulldown, curl, tricep extension, squat, deadlift, RDL, leg press, leg curl, calf raise, etc. — keep names short; primary muscle tagged.

## Acceptance
- [ ] First sign-in writes `profile`, `program/active`, and `exercises/*` documents
- [ ] Second sign-in performs zero writes (idempotency verified)
- [ ] Seeded `program/active` matches the Upper/Lower template described above
- [ ] No browser console warnings/errors on first run
