## Description
Define the canonical Firestore TypeScript schema, typed path builders, and `withConverter` helpers so every read/write through `db` is fully typed.

**Size:** M
**Files:** `src/lib/db/types.ts`, `src/lib/db/paths.ts`, `src/lib/db/converters.ts`

## Approach
- Types cover all collections in PRD section 8: `Profile`, `ProgramDoc`, `ProgramSession`, `PlannedExercise`, `Exercise`, `SessionDoc`, `LoggedSet`, `DailyDoc`, `PRDoc`.
- Path builders return `DocumentReference<T>` / `CollectionReference<T>` parameterized by the type.
- Canonical units in stored shape: `weightKg`, `bodyweightKg`, `waterMl`, `proteinG`. Convert at the display layer.
- `localDate: string` (YYYY-MM-DD) field on `DailyDoc` and `SessionDoc` (per gap analyst on tz).

## Acceptance
- [ ] All collection types compile with `strict: true`
- [ ] Path builders accept `uid: string` and require no further generics at the call site
- [ ] A sample `getDoc(profilePath(uid))` returns `Profile | undefined` with no casts
