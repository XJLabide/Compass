Write `firestore.rules`: default-deny root, per-user access scoped to `users/{uid}/...`, allowlist enforcement on `request.auth.token.email`, and per-collection field validators. Add `docs/FIRESTORE_RULES.md` with deploy + allowlist-edit instructions.

## Approach
- Allowlist in the rules file as a function returning a string list (`function allowed() { return ['email1@...', 'email2@...']; }`).
- Validators:
  - `daily`: `bodyweightKg` > 0 if present; `sleepHours` 0..24; `sleepQuality` 1..5; `proteinG` >= 0; `mood` 1..5.
  - `sessions.sets[*]`: `weightKg` >= 0, `reps` >= 0, `rpe` 1..10 if present.
  - `prs`: client cannot fabricate; only set fields the path builder writes.
- Tests use `@firebase/rules-unit-testing` to assert: unauthenticated denied; other-user denied; invalid field shapes denied; valid writes allowed.
