# Data Model — Firestore types, security rules, seeds, settings page

## Overview
Define the canonical Firestore schema in TypeScript, lock it down with security rules (per-user + allowlist + field validation), seed the default Upper/Lower program and exercise master list on first sign-in, and ship the Settings page that captures the targets and timezone the dashboard depends on.

## Scope
**In:**
- `src/lib/db/paths.ts` — typed path builders for `users/{uid}/...`
- `src/lib/db/types.ts` — TS interfaces for `profile`, `program/active`, `exercises`, `sessions`, `daily`, `prs`
- Firestore converters (`withConverter`) so reads/writes are typed end-to-end
- `firestore.rules` file in repo root + deployment notes in `docs/FIRESTORE_RULES.md`
  - Default-deny root
  - `match /users/{uid}/{document=**}` allowed if `request.auth.uid == uid` AND `request.auth.token.email in [allowlist]`
  - Per-collection write validators (bodyweight > 0, sleepHours 0..24, reps > 0, weight >= 0, RPE 1..10)
- First-run seed: on first sign-in (no `profile` doc exists), seed `profile` with defaults + `program/active` (Upper A / Lower A / Upper B / Lower B) + master `exercises` list (~25 common lifts)
- `/settings` page: unit system (imperial/metric), IANA timezone (auto-detect + dropdown), protein target g/day, weekly gain target lb/week, sign-out

**Out:**
- Program editor UI (deferred to v2 — for v1 the template is editable via direct Firestore only; UI in v2)
- PR recompute on edit (lives in E4)

## Approach
- Store canonical units: **kg for weight, ml for water, g for protein**. Unit toggle is a display-time conversion (per practice-scout / gap analyst).
- `localDate` (`YYYY-MM-DD` in the user IANA tz) is computed client-side and stored on every `daily` and `sessions` doc, alongside server timestamps. This anchors the "today" rollover.
- Allowlist for rules: hard-coded in the rules file for v1; pull from env-driven generator script if it gets unwieldy.
- Seed runs idempotently — checks `profile` existence before writing.

## Quick commands
```bash
# Deploy rules (after firebase CLI setup):
firebase deploy --only firestore:rules

# Manual verification: open /settings, change a target, reload, value persists
```

## Acceptance
- [ ] TypeScript interfaces cover all collections in PRD section 8
- [ ] Path builders return correctly typed DocumentReference<T> / CollectionReference<T>
- [ ] `firestore.rules` denies unauthenticated reads/writes
- [ ] Rules deny a signed-in user from reading another uid data (verified via emulator or manual test)
- [ ] Rules reject a `daily` write with bodyweight: -5 or sleepHours: 30
- [ ] First sign-in creates `profile`, `program/active`, and seed `exercises` documents
- [ ] Settings page reads/writes the four target fields + timezone, persists across reload
- [ ] Unit toggle changes display only, not stored values
